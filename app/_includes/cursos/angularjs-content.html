Objetivo Principal:

El Desarrollo Web en general siempre está en rápido y constante cambio, pero siempre hay un grupo de tecnologías establecidas que marcan la pauta sobre la dirección y las buenas prácticas que se implementan en el proceso. AngularJS, al ser un proyecto completamente patrocinado por Google y por su amplia adopción, está en camino de convertirse (si no es que ya lo es) en una de estas tecnologías. El Objetivo de este curso es darle un vistazo a como se hace Desarrollo Web en el 2014, enfocado principalmente en AngularJS pero sin dejar de lado tecnologías complementarias como Twitter Boostrap, GitHub, Sublime Text 2 y WebStorm.

Dirigido a:

  • Programadores Front-End con interés en ampliar sus conocimientos más allá de jQuery y la forma tradicional de comunicarse con el servidor y estructurar páginas y aplicaciones web.
  • Programadores Back-End con conocimiento del modelo MVC y con interés en conocer como AngularJS está implementando patrones de diseño en el Front-End.

Pre requisitos & preparación:

  • JavaScript basico
  • HTML
  • CSS
  • Laptop (Mac preferiblemente)

Al finalizar este curso usted habrá aprendido:

Ventajas de usar un approach MVC del lado del cliente, comparando una solución de jQuery contra lo que se puede hacer en Angular, revisión de los frameworks más famosos y porque AngularJS es la mejor solución actualmente, revisión de los sitios más importantes de desarrollo web para mantenerse actualizado.

Inicio de un proyecto web desde la iteración 0: Pasos para instalar Sublime Text, Jetbrains WebStorm, aprender a trabajar con Git desde la linea de comandos y desde el IDE, aplicando conceptos de branches, los comandos commit, push y pull. Una vez listo el entorno de desarrollo se procede (y de ahora en adelante lo mismo para cada lección) a bajar un repositorio con código para empezar a entender el concepto de Data Binding de Angular, y una revisión de la notación JSON.

Se empieza a entender como usar el $scope con ejemplos de cómo ayuda a separar la lógica de negocios de la vista, se introduce al estudiante con el concepto de controladores y de cuáles son las mejores prácticas para dividir el código por controladores. Se muestran los nuevos tags y el ciclo de vida para “enseñarle” al browser nuevos comportamientos. Uso de los filtros built-in de Angular y de como crear nuevos filtros.

Se muestra el concepto de routing en el front-end, y de como AngularJS lo implementa, cuáles son los casos más comunes a la hora de aplicarlo y el porqué ayuda a ordenar el código. Creación paso a paso de una inyección de dependencias, además de una explicación de cual es el mecanismo para que esto suceda. Concepto de directivas y de porqué son tan buenas para re-utilizar y modularizar el código, comunicación entre directivas. Todo esto mediante la modificación y explicación linea por linea de una directiva previamente creada para la clase.

Un pequeño vistazo de tecnologías complementarias a AngularJS: Twitter Bootstrap para ayudar a estructurar la vista y su soporte en el Responsive Web Design, revisión de los subproyectos de Angular UI para expandir la Vista, practicar con las funciones más comunes para manejar colecciones y objetos con Underscore.js, Scaffolding con Yeoman y su utilidad para crear la estructura de un proyecto, además de como se liga a Bower para despreocuparse del mantenimiento y búsqueda de scripts de los que depende un proyecto, y Grunt para automatización de tareas tediosas y repetitivas (pero necesarias) como minificar JS y CSS, actualización de cambios con LiveReload y su extensibilidad. Creación y automatización de unit testing con Jasmine, Angular Mocks y PhantomJS.

Validaciones de formularios y de como AngularJS automáticamente agrega comportamientos a los tags <form /> para tareas comunes, suscripción de objetos al $watch para estar viendo sus cambios de estados, $setValidity para validar o invalidar un formulario, ng-submit para capturar los eventos de submit de un formulario. Mayor separación lógica del código mediante los conceptos de service, factory y provider, y en qué caso es adecuado cada uno. Conceptos de AJAX y REST, como hacer solicitudes HTTP al servidor mediante los servicios $http y $resource. Repaso general de la aplicación creada en las lecciones, revisión del futuro próximo del desarrollo web y técnicas para estar siempre actualizado.
