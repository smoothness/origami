Objetivo Principal:

El Desarrollo Web es complejo y está en constante cambio, por ejemplo en los últimos años hemos tenido un boom en nuevos términos y técnicas para desarrollar en el browser, tales como Responsive Web Design, Single Page Applications, REST o JSON.

La lógica de negocios es una tarea que siempre le había sido delegada al servidor: lenguajes como Java, C#, PHP y Ruby eran los dominantes a la hora de programar las tareas complejas de un sitio, pero esto ha dejado de ser común y necesario, debido a que cada vez contamos con browsers con mejores capacidades y más herramientas, lo que ha causado que cada vez esta compleja lógica de negocios sea programada en el browser con frameworks y librerías tales como jQuery, AngularJS, Backbone, Sencha o famo.us, aunque también ha causado que en el servidor podamos programar con JavaScript usando tecnologías tales como NodeJS, MongoDB, Project Avatar o MeteorJS.

JavaScript es, para bien o para mal, el lenguaje de la web porque es el único lenguaje de programación que todos los browsers pueden ejecutar; todas las librerías y términos nuevos que usamos (o que vamos a aprender eventualmente por el auge que tiene el Front-End en estos momentos) van a estar directamente emparentadas con este lenguaje.

Es por estas razones que el correcto uso de JavaScript es tan importante para comprender y sacarle el mayor provecho a todas estas nuevas tecnologías basadas en un lenguaje que tiene todo el potencial de seguir vigente por un buen tiempo.

Dirigido a:

  • Programadores Front-End que estén interesados en ampliar sus conocimientos más allá de interacción básica con el DOM y sobre buenas prácticas y a ordenar código con patrones de diseno en general a la hora de usar JavaScript.
  • Programadores Back-End que quieran entender cómo se diferencia JavaScript a los lenguajes tradicionales orientados a objetos y a cómo se implementa la programación funcional y orientada a prototipos de JavaScript.
